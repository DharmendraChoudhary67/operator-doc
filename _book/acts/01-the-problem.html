When the Crown Falls

We start with leader failure because it is the fastest way to see the whole system at once.

Traffic is still flowing. The client still wants answers. Somewhere inside the cluster, replicas are listening, gossip is moving, and the system has to decide whether this is a brief interruption, a real failure, or the beginning of a failover story.

This single scene exposes the roles of Kubernetes, operators, storage, networking, and secrets.

The rest of the booklet keeps returning to three questions:

  • Who notices failure first?
  • How does the client find the right node again?
  • What makes a restarted pod count as the same member instead of a brand new one?

The Main Components

Component Real system role Why it matters
Kubernetes control plane API server, controller loops, scheduler, and resource lifecycle management keeps desired state, scheduling, and resource lifecycles moving
Operator / controller application-specific reconciliation logic encodes application-specific behavior and recovery logic
Valkey nodes running Valkey members own slots, replication, and runtime cluster state
Valkey cluster bus node-to-node gossip channel spreads topology changes and failure signals
PVC / persistent storage persistent volume claims and attached storage keeps data and cluster memory alive across pod restarts
nodes.conf local node identity file preserves node ID and peer knowledge
Client application or API client feels redirects, retries, and outages first
Secrets path Secrets, API server, etcd, kubelet controls trust material such as passwords, ACLs, and TLS

Why This Is More Than “Run a StatefulSet”

Valkey on Kubernetes is not only about keeping containers alive. It is about preserving the things that make a cluster recognizable to itself:

  • slot ownership
  • replica relationships
  • node identity
  • stable networking references
  • trusted credentials

Kubernetes can restart pods and recreate resources, but it does not naturally understand shard ownership, cluster join order, or what it means for a node to return with memory. That is the gap where operators become useful.
